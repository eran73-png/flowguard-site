נבנה על ידי מהנדס אבטחה

FlowGuard was born from a real problem — an IT professional who wanted to know exactly what his AI agent was doing on his systems.

FlowGuard נולד מבעיה אמיתית — איש IT שרצה לדעת בדיוק מה סוכן ה-AI שלו עושה על המערכות שלו.

The Problem

הבעיה

When Claude Code started running on my systems — managing files, executing commands, calling APIs — I realized there was no way to audit what it was actually doing. No log. No alert. No audit trail.

כשהתחלתי להשתמש ב-Claude Code — ניהול קבצים, הרצת פקודות, קריאות API — הבנתי שאין שום דרך לבדוק מה הוא עושה בפועל. אין לוג. אין התראה. אין שביל ביקורת.

As an IT professional with 20+ years managing enterprise infrastructure, this was unacceptable. Every change to a production system leaves a trace. Every privileged action is logged. Why should AI be different?

כאיש IT עם 20+ שנות ניסיון בניהול תשתיות ארגוניות — זה היה בלתי מקובל עליי. כל שינוי במערכת ייצור משאיר עקבות. כל פעולה מיוחסת מתועדת. למה שה-AI יהיה שונה?

"If you wouldn't let an unknown intern execute unrestricted commands on your servers, why would you let an AI agent do it?"
"אם לא היית מאפשר לעובד לא מוכר להריץ פקודות ללא הגבלה על השרתים שלך — למה שתאפשר לסוכן AI לעשות את זה?"

So I built FlowGuard. Starting as a simple hook script, it grew into a full security monitoring platform — real-time event collection, Telegram alerts, a browser dashboard, 69+ detection rules, file audit scanning, SIEM integration, and automated reporting.

אז בניתי את FlowGuard. מסקריפט hook פשוט, הוא גדל לפלטפורמת ניטור אבטחה מלאה — איסוף אירועים בזמן אמת, התראות Telegram, לוח בקרה בדפדפן, 69+ כללי זיהוי, סריקת קבצים, אינטגרציית SIEM ודיווח אוטומטי.

Phase A is monitoring. Everything FlowGuard detects is logged and alerted — but never blocked. Your AI keeps working. You stay informed. Phase B (coming) will add active blocking — automatically stopping dangerous actions before they execute.

שלב A הוא ניטור. כל מה ש-FlowGuard מזהה מתועד ומתריע — אך לעולם לא נחסם. ה-AI ממשיך לעבוד. אתה נשאר מעודכן. שלב B (בקרוב) יוסיף חסימה פעילה — עצירה אוטומטית של פעולות מסוכנות לפני שהן מתבצעות.
